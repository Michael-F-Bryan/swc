The different kinds of types recognized by the compiler

Variants

Slice(TypeSlice)

A variable-length array ([T])

Array(TypeArray)

A fixed length array ([T; n])

Ptr(TypePtr)

A raw pointer (*const T or *mut T)

Reference(TypeReference)

A reference (&'a T or &'a mut T)

BareFn(TypeBareFn)

A bare function (e.g. fn(usize) -> bool)

Never(TypeNever)

The never type (!)

Tup(TypeTup)

A tuple ((A, B, C, D, ...))

Path(TypePath)

A path (module::module::...::Type), optionally "qualified", e.g. <Vec<T> as SomeTrait>::SomeType.

Type arguments are stored in the Path itself

TraitObject(TypeTraitObject)

A trait object type Bound1 + Bound2 + Bound3 where Bound is a trait or a lifetime.

ImplTrait(TypeImplTrait)

An impl Bound1 + Bound2 + Bound3 type where Bound is a trait or a lifetime.

Paren(TypeParen)

No-op; kept solely so that we can pretty-print faithfully

Group(TypeGroup)

No-op: kept solely so that we can pretty-print faithfully

Infer(TypeInfer)

TypeKind::Infer means the type should be inferred instead of it having been specified. This can appear anywhere in a type.

Macro(Macro)

A macro in the type position.
